Getting Started

Import from tiny_harness: AgentPromptConfigToolDef are the four public exports. All other classes are instantiated internally by the Agent.

install.sh · Quick start
# Install pip install tiny-harness # Set API key export ANTHROPIC_API_KEY="sk-ant-..." # Run one-shot tiny-harness "Create hello.py" # Session mode tiny-harness --model claude-sonnet-4-20250514 --skills files # TUI mode pip install tiny-harness[tui] tiny-harness --tui --skills files
python · Python API minimal example
from tiny_harness import Agent, Prompt, Config import os agent = Agent( prompt=Prompt("You are a helpful assistant."), config=Config( model="deepseek-chat", api_key=os.environ["DEEPSEEK_API_KEY"], workspace=".", ) ) agent.load_skill("files") result = await agent.run("Create hello.py")
class Agent

Agent

The main orchestrator. Creates all components at init — LLM provider, tool registry, filesystem guard, event bus, message manager. The only class users construct directly.

Constructor: Agent(prompt: Prompt, config: Config, store: SessionStore | None = None)

MethodReturnsDescription
__init__(prompt, config, store=None) — Creates the agent. Builds LLMProvider, ToolRegistry, ToolExecutor, FilesystemGuard, EventBus, MessageManager internally. Optional store for session persistence.
run(prompt: str) str Execute a single prompt synchronously (async). Returns the LLM's final text response. Raises RuntimeError if already running.
run_stream(prompt: str) AsyncIterator[StreamEvent] Stream execution events as they happen. Yields StreamEvent objects for text deltas, tool calls, errors, and iteration progress.
load_skill(skill_ref: str) None Load a skill by name ("files"), Python module path, or file path. Resolves via tiny_harness.skills.{name} → import → file. Idempotent: loading twice is a no-op.
clear() None Reset the conversation history. Retains the system prompt (first message).
start_session(session_id=None) str Start a new persistence session. Returns the session ID. Creates a SessionStore if none was provided at init.
on(event_type: str, handler) None Subscribe to events matching event_type. Handler receives a StreamEvent. Supports both sync and async handlers.
PropertyTypeDescription
toolsToolRegistryAccess the tool registry to register custom tools.
eventsEventBusDirect access to the event bus for advanced subscription patterns.
session_idstr | NoneThe current session ID if a session was started.
storeSessionStore | NoneThe session store if one is active.
@dataclass class AgentConfig

Config (alias: AgentConfig)

All agent configuration in one dataclass. Exported as Config from the public API.

Import as from tiny_harness import Config. Underlying class is AgentConfig.

FieldTypeDefaultDescription
modelstrrequiredModel identifier (e.g. "claude-sonnet-4-20250514", "gpt-4o", "deepseek-chat")
api_keystrrequiredProvider API key
workspacestrrequiredRoot directory for file operations. All file paths are resolved relative to this.
providerstr"deepseek"LLM provider: "anthropic", "openai", or "deepseek"
api_base_urlstr | NoneNoneCustom API endpoint. Defaults to provider standard (only used for openai/deepseek).
max_iterationsint25Maximum loop iterations before forcing a degraded finish.
max_errorsint10Total tool error budget across all iterations.
max_consecutive_errorsint3Consecutive tool errors before aborting.
timeout_msint30_000Per-tool execution timeout in milliseconds (30s default).
max_tool_result_charsint50_000Truncate tool results exceeding this character count.
python · Config examples
# Minimal config config = Config(model="claude-sonnet-4-20250514", api_key="sk-...", workspace=".") # DeepSeek with custom endpoint config = Config( model="deepseek-chat", api_key="sk-...", workspace=".", provider="deepseek", api_base_url="https://api.deepseek.com/v1", ) # OpenAI with safety tuning config = Config( model="gpt-4o", api_key="sk-...", workspace=".", provider="openai", max_iterations=10, timeout_ms=15_000, )
class Prompt

Prompt

First-class artifact for building the system prompt. Separate from Config (ADR 002). Supports compositional section appending.

MethodReturnsDescription
__init__(base: str)—Create a prompt with a base section. This becomes the first section.
append(section: str)NoneAdd a new section. Sections are joined with "\n\n" when serialized.
to_string()strJoin all sections with double newlines. Called internally by MessageManager to produce the system message.
python · Prompt composition
prompt = Prompt("You are a helpful assistant.") prompt.append("Always use tools when you need to access files.") prompt.append("Be concise in your responses.") # Result: "You are...\n\nAlways use...\n\nBe concise..."
@dataclass class ToolDef

ToolDef

Defines a tool's contract: name, description, JSON Schema parameters, and risk level for filesystem guarding.

FieldTypeDefaultDescription
namestrrequiredUnique tool identifier. Used by LLM to request tool calls.
descriptionstrrequiredHuman-readable description sent to the LLM as tool instructions.
parametersdictrequiredJSON Schema object describing expected arguments. Must have "type": "object" with "properties" and "required".
risk_levelstr"read_only"Filesystem risk: "safe" (no guard), "read_only" (read check), "mutation" (write check), "destructive" (delete check).
python · Custom tool registration
from tiny_harness import ToolDef # Define a tool weather_def = ToolDef( name="weather", description="Get current weather for a city.", parameters={ "type": "object", "properties": {"city": {"type": "string"}}, "required": ["city"], }, risk_level="safe", ) # Register with agent agent.tools.register_from_def(weather_def, handler=lambda args: f"Sunny, 22°C in {args['city']}")

class ToolExecutor

ToolExecutor

Executes tool calls: validates arguments against JSON Schema, enforces filesystem guard, runs the handler with timeout, formats results.

MethodReturnsDescription
execute(name, args, call_id)ToolResultValidate schema → guard workspace → execute handler (with timeout) → format output. Returns ToolResult with success/error status.
get_definitions()list[dict]Proxy to ToolRegistry.get_definitions().

Execution pipeline: name lookup → JSON Schema validation → filesystem guard → handler(args) → output formatting. Typo suggestions via difflib.get_close_matches. Timeout via asyncio.wait_for. Results exceeding max_tool_result_chars are truncated with a note.

Sync Handler
def my_tool(args: dict) -> str: return f"Hello {args['name']}"
Async Handler
async def my_tool(args: dict) -> str: result = await fetch_data(args['url']) return result

ToolResult

Return value from tool execution. Success/error status with content.

FieldTypeDescription
successboolWhether the tool executed successfully.
tool_call_idstrThe LLM-assigned tool call ID for matching results to requests.
contentstrResult text or error message.
Class MethodReturnsDescription
ok(call_id, content)ToolResultCreate a success result.
error(call_id, message)ToolResultCreate an error result. The message is sent back to the LLM for self-correction.
class AnthropicProvider(LLMProvider)

AnthropicProvider

Native Anthropic API provider. Uses SSE streaming, handles tool use content blocks, implements exponential backoff retry.

MethodReturnsDescription
__init__(api_key, model, retry_config=None)—Create provider. Optional LLMRetryConfig for custom retry behavior.
generate(messages, tools=None)LLMResponseNon-streaming call. Converts messages to Anthropic format (extracts system). Retries on 429/529/5xx with exponential backoff.
generate_stream(messages, tools=None)AsyncIterator[LLMStreamChunk]SSE streaming. Yields text_delta, tool_call_start, tool_call_delta, tool_call_end chunks.

Message conversion: System messages extracted into the system field (Anthropic API requirement). Tool definitions converted to Anthropic format with input_schema. Max tokens: 16,384.

class OpenAIProvider(LLMProvider)

OpenAIProvider

OpenAI-compatible API provider. Works with OpenAI, DeepSeek, and any OpenAI-compatible endpoint. Buffers streaming tool call JSON deltas.

MethodReturnsDescription
__init__(api_key, model, base_url, retry_config=None)—Create provider with custom base URL. Default: "https://api.openai.com/v1".
generate(messages, tools=None)LLMResponseNon-streaming call. Converts tools to {"type": "function", "function": {...}} format. Same retry logic as Anthropic.
generate_stream(messages, tools=None)AsyncIterator[LLMStreamChunk]SSE streaming. Buffers tool call JSON fragments (tool_buffers) and emits complete ToolCallRequest at tool_call_end.
@dataclass

LLM Types

Data types used across the LLM abstraction layer.

LLMResponse
  • text str | None — text response
  • tool_calls list[ToolCallRequest]
  • usage TokenUsage — input/output tokens
  • finish_reason str — stop reason
  • is_final() — True when no tool calls
ToolCallRequest
  • id str — LLM-assigned call ID
  • name str — tool name
  • arguments dict — parsed arguments
LLMStreamChunk
  • type str — chunk type
  • content str | None — text/json
  • tool_call ToolCallRequest | None
LLMRetryConfig
  • max_retries int = 3
  • base_delay float = 1.0s
  • max_delay float = 60.0s
  • backoff_factor float = 2.0
TokenUsage
  • input_tokens int
  • output_tokens int
@dataclass(frozen=True) class StreamEvent

StreamEvent & EventBus

Immutable event objects emitted during agent execution. Subscribe via agent.on() or agent.events.subscribe().

text_delta
Partial text from LLM streaming. content contains the fragment.
tool_start
Tool execution begins. tool_name and content (JSON args).
tool_end
Tool execution complete. tool_name and content (result preview, first 100 chars).
iteration
Loop iteration tick. num (current iter), max (max iters), content (token count).
error
Error condition. message contains the error description.
end
Agent finished. content contains final answer text.
python · Event subscription patterns
# Pattern 1: agent.on() convenience method agent.on("tool_start", lambda e: print(f"→ {e.tool_name}")) agent.on("error", lambda e: print(f"✗ {e.message}")) # Pattern 2: direct EventBus access async def handle_all(event): print(f"[{event.type}] {event.content or ''}") agent.events.subscribe(handle_all) # Pattern 3: streaming with async for async for event in agent.run_stream("Create hello.py"): if event.type == "text_delta": print(event.content, end="", flush=True)
class FilesystemGuard

FilesystemGuard

Workspace boundary enforcement. Prevents path traversal and access outside the configured workspace.

MethodReturnsDescription
__init__(workspace: str)—Store the resolved absolute workspace path.
resolve(path: str)strExpand ~, resolve relative to workspace, normalize via os.path.realpath. Raises PathAccessError on null bytes or resolution failure.
guard(path, operation)strResolve path and verify it's within workspace. Raises PathAccessError if outside. Returns resolved path.

Risk levels: safe tools skip the guard entirely. read_only, mutation, and destructive tools pass through the guard before execution. The PathAccessError is caught by ToolExecutor and returned as a ToolResult.error.

class SessionStore

SessionStore

JSONL-based session persistence. Stores conversation turns in ~/.tiny-harness/sessions/ by default.

MethodReturnsDescription
__init__(base_dir=None)—Create store. Default path: "~/.tiny-harness/sessions/".
new_session()strGenerate a 12-char hex session ID.
save_turn(session_id, chat_id, role, ...)NoneAppend a JSON record to the session file. Supports content, tool_calls, tool_results, token_usage, model, iteration, metadata.
load_session(session_id)list[dict]Read all turns for a session from the JSONL file.
list_sessions()list[dict]List all sessions sorted by modification time. Each entry has session_id, created, updated, turns, model, file.
delete_session(session_id)boolDelete a session file. Returns True if deleted.
export_session(session_id)dictExport session as a dict with all records for serialization.
python · Session persistence
from tiny_harness import Agent, Prompt, Config from tiny_harness._persist import SessionStore store = SessionStore() agent = Agent( prompt=Prompt("You are helpful."), config=Config(model="deepseek-chat", api_key="sk-...", workspace="."), store=store, ) agent.start_session() # returns session ID await agent.run("Hello") # automatically saved # Later: list sessions for s in store.list_sessions(): print(f"{s['session_id']}: {s['turns']} turns, model={s['model']}")

SKILLS
Built-in Skills

Skills are packaged bundles of tools + prompt instructions. Load with agent.load_skill("name"). Each skill has a register(agent) function that wires its tools and prompt sections into the agent.

files (default in CLI)
  • read_file — read with offset/limit
  • write_file — create or overwrite
  • list_directory — with pattern/recursive
  • find_files — glob search
  • delete_file — remove file
  • create_directory — mkdir -p
  • move_file — rename/move
shell
  • execute_command — run shell command with timeout
  • environment_info — OS, Python version, env vars
  • Risk level: mutation
search
  • grep — regex search across files
  • ast_grep — AST-aware pattern search
  • glob — file pattern matching
  • read — file reading
  • lsp_symbols — symbol lookup
  • Risk level: read_only
python · Loading skills
# Load a single skill agent.load_skill("files") # Load multiple skills agent.load_skill("files") agent.load_skill("shell") agent.load_skill("search") # Load from a Python module path agent.load_skill("my_package.my_skill") # Load from a file path agent.load_skill("./custom_skill.py") # Idempotent: loading same skill twice is a no-op agent.load_skill("files") # no error, silently skipped
